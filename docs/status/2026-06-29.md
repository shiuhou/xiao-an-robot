Wiring under test:

- Shared BCLK: GPIO39
- Shared WS/LRC: GPIO40
- INMP441 SD/DOUT: GPIO41
- MAX98357A DIN: GPIO47
- INMP441 VDD: 3V3, L/R: GND
- MAX98357A VIN: 5V

What changed:

- Added `mergetesting_audio_shared_i2s_diag`.
- Added standalone `src/audio_shared_i2s_diag_main.cpp`; it does not run the normal `MergetestingApp`, Camera, TFT, Motor, WiFi, WebSocket, or OpenClaw path.
- The diagnostic uses half-duplex only and never keeps RX/TX drivers installed at the same time:
  - LISTEN: install RX on I2S0, BCLK=39, WS=40, DIN=41; print RMS/peak/voice activity and health metrics.

  - SPEAK: install TX on I2S1, BCLK=39, WS=40, DOUT=47; play a 1 kHz probe tone first, then embedded `I can speak now.` at gain 20.
  - STOP_SPEAK: stop/uninstall TX and hold GPIO47 LOW.
- The source has a compile-time guard against GPIO35/36/37 in this diagnostic.

## INMP441 COM22 mic shift/channel diagnostic

Scope: COM22 mic-only bring-up with current wiring BCLK=39, WS=40, MIC SD=41,
speaker disabled because the speaker module is currently unavailable.

What changed:

- `mic_stream.cpp` now keeps the old behavior as defaults but exposes:
  - `MERGETEST_MIC_SHIFT_BITS` default `14`
  - `MERGETEST_MIC_CHANNEL_FORMAT` default `I2S_CHANNEL_FMT_ONLY_LEFT`
- Added diagnostic envs:
  - `mergetesting_mic_only_shift16`
  - `mergetesting_mic_only_right_shift16`

Evidence:

- Original `mergetesting_mic_only` (`>>14`, left) produced an unintelligible noisy
  5s WAV: RMS `-26.48 dBFS`, peak `-15.28 dBFS`, DC offset `2.81%`; user heard
  only loud noise.
- High-pass 80Hz reduced low-frequency energy but the user still heard only
  noise, so the fault is not only ASR/VAD or low-frequency filtering.
- `mergetesting_mic_only_shift16` uploaded to COM22 and produced
  `runtime/manual_samples/mic_com22_shift16_5s.wav`: RMS `-37.51 dBFS`, peak
  `-23.00 dBFS`, DC offset `0.79%`.
- `mergetesting_mic_only_right_shift16` uploaded to COM22 and produced
  `runtime/manual_samples/mic_com22_right_shift16_5s.wav`: all zero PCM.

Conclusion:

- With INMP441 L/R tied to GND, left channel is the correct channel; right
  channel is silent.
- `>>14` is too hot/noisy for this module/path. Use `>>16` as the next mic
  diagnostic baseline before ASR/VAD tuning.
- The board was restored to `mergetesting_mic_only_shift16` on COM22 after the
  comparison.
