# 2026-06-29 Status

## INMP441 raw audio diagnostics

Scope: base-station `/audio` observability before ASR tuning.

What changed:

- Added `base_station.perception.audio_diagnostics` to inspect raw `pcm_s16le` mic captures.
- `python -m base_station.perception.audio_diagnostics runtime\latest_audio.pcm --wav-out runtime\manual_samples\mic_20cm.wav --report-out runtime\manual_samples\mic_20cm_stats.json` converts the latest PCM window to WAV and writes RMS/peak/DC/clipping stats.
- `base_station/ws_server/server.py` now writes the same latest-window stats into `runtime/audio_stats.json` under `latest_window` whenever `/audio` receives PCM.

Use this before changing ASR/VAD/model settings. If the 20cm WAV is unclear, debug I2S format, INMP441 32-bit to 16-bit shift, gain, sample rate, channel, byte order, power, wiring, or acoustics first. If the WAV is clear and ASR is bad, then debug ASR, language settings, VAD segmentation, or frontend preprocessing.

## Speaker spoken-TTS chain

Scope: `robot/mergetesting` speaker-only and phrase-only diagnostics on COM19.

What is confirmed:

- COM19 upload works.
- Hotspot/WS came back later with robot IP `192.168.137.200` and host `192.168.137.1`.
- Safe local audio path is still good: serial `sound wakeup_chime` logged `[Speaker] play_local done wakeup_chime ok=true`, and the user confirmed it can make sound.
- `mergetesting_speaker_phrase_only` can be built with embedded sentence PCM `I can speak now.`.
- After rewiring MAX98357A to BCLK=GPIO39, LRC/WS=GPIO40, DIN=GPIO41, `mergetesting_speaker_altpins_phrase_only` completed the full embedded sentence PCM write without WDT.
- The board was restored to `mergetesting_speaker_altpins_only` over COM19 after the phrase diagnostic, so the current flashed env matches the temporary GPIO39/40/41 speaker wiring and does not auto-speak.

Failure captured:

- Serial `tts serial` reached `speaker_play_tts_mock()`.
- The firmware logged `tts embedded phrase bytes=52000 text=I can speak now. preview=serial`.
- It then entered `pcm mono write start bytes=52000 frames=26000 timeout_ticks=50`.
- Before any PCM progress log, the board reset with `rst:0x8 (TG1WDT_SYS_RST)`.
- This proves the command path reaches the speaker task; the remaining failure is inside PCM-to-I2S playback, not OTA or WebSocket routing.

No-sound code change prepared:

- `writeMonoPcmS16Le()` now trims leading quiet PCM frames before the first I2S write.
- The embedded phrase has about 95-101 ms of leading silence, so the next audible test should start I2S with non-zero speech data instead of an all-zero first buffer.
- GPIO35/36/37 are now the leading hardware suspect on ESP32-S3 Octal PSRAM boards. Added speaker-only A/B envs that move MAX98357A to GPIO39/40/41:
  - `mergetesting_speaker_altpins_only` for serial `sound wakeup_chime`.
  - `mergetesting_speaker_altpins_phrase_only` for serial `tts serial`.
- GPIO39/40/41 are the repo's default INMP441 mic pins, so these envs are diagnostic speaker-only targets, not a permanent combined mic+speaker map.
- Added OTA upload targets for the same diagnostic wiring so the board can be powered from external 5V without USB:
  - `mergetesting_speaker_altpins_only_ota`
  - `mergetesting_speaker_altpins_phrase_only_ota`

GPIO39/40/41 A/B result:

- `mergetesting_speaker_altpins_only` upload over COM19: PASS.
- Serial `sound wakeup_chime` on alt pins: PASS, log `runtime/speaker_altpins_sound_com19_20260629_161251.log`, no WDT.
- External 5V/no-USB path: PASS for WebSocket `audio.play_local wakeup_chime`; OTA/WS used robot IP `192.168.137.200`, user confirmed sound.
- `mergetesting_speaker_altpins_phrase_only` upload over COM19: PASS.
- Serial `tts serial` on alt pins first passed at firmware level with the wiring later found incorrect, log `runtime/speaker_altpins_phrase_tts_com19_20260629_162330.log`.
- After the user corrected the wiring, serial `tts serial` passed again at firmware level, log `runtime/speaker_altpins_phrase_tts_correct_wiring_com19_20260629_162737.log`.
- External 5V/no-USB phrase path: `mergetesting_speaker_altpins_phrase_only_ota` OTA upload PASS, WebSocket `audio.play_tts` forwarded PASS, then `mergetesting_speaker_altpins_only_ota` restored PASS.
- Key log evidence: `tts embedded phrase bytes=52000 text=I can speak now.`, `pcm mono skipped leading quiet frames=1620`, `pcm mono write complete frames=26000`, `tts embedded i2s released`, `tts mock done ok=true`.
- Conclusion: the previous 35/36/37 path fails at PCM-to-I2S first write, while 39/40/41 completes the same PCM path. Treat GPIO35/36/37 as unusable for MAX98357A on the current ESP32-S3 Octal PSRAM module.
- Remaining user-facing check: whether the phrase was audibly heard during the external 5V WebSocket `audio.play_tts` run.
