# Status 2026-06-30

## INMP441 fixed-window ASR calibration

Scope: fixed-window microphone command demo before full real-time ASR.

Hardware state:

- INMP441 only; MAX98357A speaker module is currently unavailable and was not used.
- Active mic wiring: BCLK/SCK GPIO39, WS/LRCLK GPIO40, SD/DOUT GPIO41, L/R tied to GND.
- Active firmware env for mic ASR calibration: `mergetesting_mic_only_shift18_asr`.

Results:

- `mergetesting_mic_only_shift16_asr` captured voice but clipped on close repeated speech: peak `0.0 dBFS`, clipping about `3.5%`; do not use as the current ASR demo gain.
- `mergetesting_mic_only_shift18_asr` uploaded successfully on COM19 and produced usable live speech levels: peak about `-12 dBFS`, clipping `0%`.
- Raw 5 second fixed-window ASR on `runtime/manual_samples/mic_com19_ready_phrase_shift18_live_20260630_023431.wav` returned a mixed transcript including wake-word-like text and repeated `你好`.
- Adding `--trim-speech` cropped that 5 second window to a 2160 ms speech segment and SenseVoice returned the focused transcript `你好你好你好你好你好你好你好`.

Software changes:

- Added `base_station/perception/audio_segments.py` for simple energy-based WAV speech trimming.
- Added `--trim-speech`, `--speech-trim-threshold`, `--speech-trim-padding-ms`, and related options to `base_station.monitor.asr_runtime`.
- Added tests in `tests/unit/test_audio_segments.py` and `tests/unit/test_asr_runtime.py`.

Demo direction:

- Short term: fixed time window, user speaks a command such as `帮我记一下，星期三下午有个会议`, then base station trims speech and runs ASR.
- Final target: real-time microphone stream, VAD/ASR detects keywords or intent, then action routing triggers robot/agent behavior.
